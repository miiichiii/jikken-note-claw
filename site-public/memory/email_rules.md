# メール運用ルール

## 優先・無視リスト
- `facebook`, `change`, `LinkedIn`, `rakuten` を含む件名／送信者は低優先／無視対象にする。
- 「予約の受付（筑波大学オープンファシリティー推進機構）」のような案内も無視でよい。
- それ以外の `hamamichi@md.tsukuba.ac.jp` 宛メールは重要度が高い可能性があるため、積極的にチェックする。
- **最優先チェック対象（漏れ厳禁）**:
    - **スレッド形式の未読メッセージ**: スレッドの途中に埋もれている個別の未読分も必ず一通ずつ内容を確認すること。
    - **重要ラベル (`IMPORTANT`) または件名に「緊急」「至急」「IMPORTANT」を含むもの**。
    - **特定のキーワードを含むもの**: 「AMED」「退官」「見積」「注文」「予算」「公募」「採択」「転送」。
    - **件名が「転送」となっているもの**: これらは大学（IPU）のグループウェアから転送された重要な行政・事務連絡であるため、最優先で中身を確認すること。
    - **学術・研究関連のドメイン (`.ac.jp`, `.or.jp`, `.gov`) からの、個人宛または特定のプロジェクトに関する連絡**。
    - **自分自身 (`hamamicchi@gmail.com`) から転送されてくるメール**: これらは `hamadami@ipu.ac.jp` (茨城県立医療大学) 宛の重要な連絡であるため、必ずチェックすること。

## 監視ルール
- **毎朝07:00 JST** に当日＋翌日の予定を確認して必要な報告を行う。
- **毎時** 、Google Gmail で未読メッセージをチェック。
- **継続報告ルール**: 重要度の高いメール（最優先チェック対象に該当するもの）が**未読のまま残っている場合、既報であっても毎回のチェックで「未読の重要案件」として繰り返し報告すること**。
- 検索には必ず `gog gmail messages search "is:unread newer_than:7d"` を使用し、過去1週間分の未読メッセージを漏らさず取得・評価すること。
- また、`from:hamamicchi@gmail.com` で検索し、茨城県立医療大学 (`ipu.ac.jp`) から転送された重要な未読メールも漏らさず取得すること。
- 重要メールがあれば内容を精査し、70Bサブエージェント（または状況に応じたモデル）を用いて詳細な要約（3〜5文）を作成してTelegramへ報告する。該当なしは報告なし。

## サブエージェントとの連携
- メール専用サブエージェントは表示された要件を毎回この `memory/email_rules.md` で再確認し、最新のルールを使って判断すること。
- メールチェックの出力には、**件名・差出人・簡単な要約**を必ず含めるようにする。
- 特に重要な依頼（件名に「IMPORTANT」が含まれるものや優先度の高い差出人）については、**本文の内容を取得して詳細な概要（3〜5文程度）**を併せて報告すること。
- 表示形式は、テレグラムでの視認性を考慮し、**表（Table）形式を避け、マークダウンの箇条書き（Bullet points）**を使用すること。
- 返信案が必要な場合はその旨を明記して送る。

## 今後のルール追加
- メールに関する新しいルールや優先する条件はすべてこのファイルに追記すること。センセイからの追加指示があれば、内容を更新するのでその指示を伝えてください。
